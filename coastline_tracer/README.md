# CoastlineTracer - 海岸线优先级追踪 QGIS 插件

> 基于优先级的海岸线自动追踪与连接工具，在三类线要素数据中按优先级自动搜索 A→B 最优路径。

## 📖 插件简介

CoastlineTracer 专为海岸线数据处理工程师设计，能够在三类线要素数据中自动寻找并连接从 A 点到 B 点的最优路径，自动优先选择海岸线，在海岸线断裂处才自动跳转到次优先级数据源。

**功能亮点：**
- 🚀 高性能图构建（空间裁剪 + 并查集合并 + KD-Tree 查询）
- 🔀 双向 Dijkstra / A* 路径搜索算法
- 🗺️ 地图交互点选 A/B 点，实时显示标记
- 🎨 追踪结果按数据来源自动分色渲染
- 📊 详细追踪统计（长度、占比、耗时）
- 💾 多格式导出（SHP/GeoJSON/GeoPackage/KML/CSV）
- ⚙️ 参数持久化，下次打开自动恢复

## ✨ 功能特性

### 🏎️ 性能特性
| 特性 | 说明 |
|---|---|
| 空间裁剪 | 根据 A/B 点自动计算 BoundingBox，只加载相关区域要素 |
| 并查集节点合并 | 使用空间哈希网格 + 并查集，O(n) 复杂度合并近邻节点 |
| KD-Tree 查询 | 最近节点查找时间复杂度 O(log n) |
| 双向 Dijkstra | 从 A/B 两端同时搜索，速度提升约 2x |
| A* 算法 | 使用 Haversine 距离作为启发函数进一步加速 |
| 图缓存 | 支持 pickle 序列化缓存，相同数据不需要重建图 |
| 后台线程 | 图构建和路径追踪均在后台线程执行，UI 不阻塞 |

### 🎨 交互特性
- 地图点击选取 A/B 点，A 点显示红色标记，B 点显示蓝色标记
- 坐标粘贴功能（支持 "lon,lat" 和 "lon lat" 格式）
- 实时计算并显示 A-B 直线距离
- 一键交换 A/B 点
- 三种预设模式快速切换（快速/标准/精确）
- 追踪完成后自动切换到结果标签页

### 📊 输出特性
- 追踪结果作为临时内存图层添加到项目
- 按来源图层（coast/build_coast/land_border）分色渲染
- 详细统计信息（总长度、各来源占比、线段数量、耗时）
- 六种导出格式：SHP、GeoJSON、GeoPackage、KML、CSV、WKT 剪贴板

## 📥 安装方法

### 方法一：从 GitHub 下载 ZIP 安装（推荐）

1. 下载本仓库 ZIP 文件：
   ```
   https://github.com/liccgithub/coastline-tracer/archive/refs/heads/main.zip
   ```
2. 打开 QGIS，菜单栏 → **插件** → **管理并安装插件**
3. 点击**从ZIP安装**，选择下载的 ZIP 文件
4. 点击**安装插件**

### 方法二：手动复制到插件目录

1. 下载或克隆本仓库
2. 将 `coastline_tracer/` 目录复制到 QGIS 插件目录：
   - **Windows**: `%APPDATA%\QGIS\QGIS3\profiles\default\python\plugins\`
   - **macOS**: `~/Library/Application Support/QGIS/QGIS3/profiles/default/python/plugins/`
   - **Linux**: `~/.local/share/QGIS/QGIS3/profiles/default/python/plugins/`
3. 在 QGIS 插件管理器中启用 **CoastlineTracer**

## 🚀 快速开始（5步完成首次追踪）

1. **加载数据**：在 QGIS 中加载 coast.shp、build_coast.shp、land_border.shp
2. **打开插件**：菜单 → 矢量 → 海岸线追踪 → 海岸线追踪器（或 `Ctrl+Shift+T`）
3. **选择图层**：在"数据与参数"标签页中选择对应图层
4. **设置起止点**：在"起止点"标签页中点击"📍 从地图选取"选取 A、B 点
5. **构建图并追踪**：点击"🔨 构建网络图"，完成后点击"🚀 开始追踪"

## 📋 详细使用步骤

### 标签页 1：数据与参数

#### 数据输入
- **海岸线图层**：选择 coast.shp 对应的图层，设置权重（默认 1.0，越低优先级越高）
- **建设线图层**：选择 build_coast.shp 对应的图层，设置权重（默认 10.0）
- **陆地边界图层**：选择 land_border.shp 对应的图层，设置权重（默认 100.0）
- 插件启动时会自动检测项目中名称含 "coast"、"build"、"border" 的图层并预选

#### 参数设置
- **节点合并容差**：距离小于该值的线端点将被合并为同一节点（单位：米）
  - 过大会降低路径精度，过小会导致图不连通
  - 建议值：10m（精确）/ 50m（标准）/ 100m（快速）
- **搜索算法**：
  - 双向Dijkstra（推荐）：速度最快，适合大多数场景
  - A*（启发式加速）：在地理距离较大时更快
  - 标准Dijkstra：基础算法，稳定可靠
- **预设模式**：一键切换参数组合

#### 空间裁剪
- 启用后，只加载 A/B 点包围盒内的要素，显著提升全球数据的处理速度
- **缓冲区扩展**：在 A/B 点包围盒基础上额外扩展的百分比，防止路径超出范围

### 标签页 2：起止点

#### 设置方法
1. **从地图选取**：点击按钮后在地图上单击，自动获取坐标
2. **手动输入**：直接在经度/纬度输入框中输入数值
3. **粘贴坐标**：复制坐标文本后点击"📋 粘贴坐标"
   - 支持格式：`121.47, 31.23`、`121.47 31.23`

#### 快捷操作
- **🔃 交换 A/B**：一键交换起点和终点
- **A-B 直线距离**：实时显示 A/B 两点的球面直线距离

### 标签页 3：追踪结果

#### 统计摘要
追踪完成后自动显示：
- 总路径长度（km）
- 各来源图层占比（百分比 + 长度）
- 线段数量
- 追踪耗时

#### 导出
| 按钮 | 格式 | 说明 |
|---|---|---|
| 💾 导出 Shapefile | .shp | ESRI Shapefile 格式 |
| 💾 GeoJSON | .geojson | GeoJSON 格式 |
| 💾 GeoPackage | .gpkg | OGC GeoPackage 格式 |
| 💾 KML | .kml | Google KML 格式 |
| 📊 导出 CSV | .csv | 坐标序列 + 属性表 |
| 📋 复制坐标 | 剪贴板 | WKT 格式 |

### 底部区域

- **🔨 构建网络图**：将三类线要素构建为加权图
- **🚀 开始追踪**：在图中搜索 A→B 最优路径
- **🗑 清除结果**：移除地图上的追踪结果图层
- **进度条**：显示图构建/路径追踪的实时进度
- **日志**：带时间戳的处理日志

## 🔬 算法原理

### 图构建过程

```
1. 遍历三个图层的所有线要素
2. 提取每条线的顶点序列
3. 相邻顶点间形成一条边，赋予成本 = 几何长度(米) × 图层权重
4. 使用空间哈希网格 + 并查集合并距离小于容差的节点
5. 构建邻接表（无向图）
```

**优先级权重机制：**
- coast.shp 边的成本 = 长度 × 1.0（最小，最优先）
- build_coast.shp 边的成本 = 长度 × 10.0
- land_border.shp 边的成本 = 长度 × 100.0

算法自然偏好成本最小的路径，即优先沿海岸线行进。

### 路径搜索过程

```
1. 用 KD-Tree 查找距离 A/B 点最近的图节点
2. 执行路径搜索算法（双向Dijkstra/A*/Dijkstra）
3. 将路径节点序列转换为边序列
4. 生成 QgsVectorLayer 并应用分色渲染
```

**双向 Dijkstra：**
- 从起点和终点同时向中间搜索
- 当两端搜索在某节点相遇时停止
- 相比单向 Dijkstra 减少约 50% 的搜索节点

**A* 算法：**
- 在 Dijkstra 基础上添加启发函数 h(n) = Haversine(n, 终点)
- 引导搜索方向朝向终点，减少不必要的探索

## ⚙️ 参数说明

| 参数 | 范围 | 默认值 | 说明 |
|---|---|---|---|
| 海岸线权重 | 0.1 ~ 9999 | 1.0 | 权重越低，优先级越高 |
| 建设线权重 | 0.1 ~ 9999 | 10.0 | 默认比海岸线低 10 倍 |
| 陆地边界权重 | 0.1 ~ 9999 | 100.0 | 默认比海岸线低 100 倍 |
| 节点合并容差 | 1 ~ 10000 m | 50 m | 节点吸附距离 |
| 缓冲区扩展 | 5 ~ 200 % | 20 % | 空间裁剪扩展量 |

### 预设模式参数对比

| 模式 | 容差 | 缓冲区 | 适用场景 |
|---|---|---|---|
| 快速 | 100 m | 30 % | 初步探索、大范围数据 |
| 标准 | 50 m | 20 % | 日常使用（推荐） |
| 精确 | 10 m | 10 % | 高精度需求 |

## ❓ 常见问题 FAQ

**Q1: 为什么没有检测到我的图层？**
> 图层组合框只显示线要素图层。如果你的图层是面或点要素，需要先转换为线要素。

**Q2: 追踪结果提示"无法找到连通路径"怎么办？**
> 可尝试以下操作：
> 1. 增大节点合并容差（如从 50m 增大到 200m）
> 2. 禁用空间裁剪或增大缓冲区百分比
> 3. 确认 A/B 点在数据覆盖范围内
> 4. 降低低优先级图层的权重

**Q3: 构建图时进度卡住了怎么办？**
> 这通常是因为数据量过大。建议：
> 1. 启用空间裁剪
> 2. 切换到"快速模式"（容差 100m）
> 3. 先缩放到 A/B 点区域，减少加载范围

**Q4: 导出的 Shapefile 乱码怎么处理？**
> 导出文件使用 UTF-8 编码。在 QGIS 中打开时，请在图层属性中将编码设置为 UTF-8。

**Q5: 追踪结果中路径颜色如何区分？**
> - 🔵 蓝色线段 → 来自海岸线（coast）
> - 🟠 橙色线段 → 来自建设线（build_coast）
> - 🟤 棕色线段 → 来自陆地边界（land_border）

**Q6: 如何查看路径的详细属性？**
> 追踪结果图层添加到项目后，使用 QGIS 的"识别要素"工具单击线段，可查看 segment_id、source_layer、length_m、priority、cost 等属性。

**Q7: 能否同时使用两个图层（不使用第三个）？**
> 可以。将不需要的图层组合框设置为空（选择"无"），插件只使用已选择的图层构建图。

**Q8: 追踪结果图层关闭后如何恢复？**
> 结果图层是临时内存图层，关闭后无法从项目中恢复。建议追踪完成后立即导出。可以重新执行追踪操作（已构建的图会保留在内存中，无需重建）。
